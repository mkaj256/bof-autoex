## Описание

Автоматизированный генератор эксплойтов для уязвимостей переполнения буфера в 64-битных ELF-файлах. Программа анализирует целевой бинарник через GDB, вычисляет смещения в стеке, генерирует позиционно-независимый шеллкод без «плохих» байтов и формирует готовую полезную нагрузку со скриптом запуска. Предназначен для учебных целей и CTF-задач, где отключены механизмы защиты (канарейки, PIE, NX).

## Возможности

- Генерация шеллкода трёх типов: `write` (вывод строки), `execve` (выполнение команды), `crash` (аварийное завершение программы с ошибкой segmentation fault)
- Автоматический обход запрещённых байтов (`\x00`, `\x0a`) за счёт приёмов работы с регистрами
- Уточнение адресов возврата через анализ core-дампа для повышения надёжности эксплуатации
- Полностью автоматизированный сбор данных через GDB — без ручной отладки
- Генерация скрипта `exploit.sh` с отключённым ASLR через `setarch -R`
- Режим тестирования: сборка автономного исполняемого файла для проверки шеллкода

## Образовательная ценность

Проект создан с акцентом на прозрачность и понятность кода. Изучая исходники, можно разобраться:

- В устройстве стека и механизме перезаписи адреса возврата (`modules/get_data.py`)
- В методах генерации шеллкода и обхода ограничений на байты (`modules/generate_shellcode_submodules/`)
- В автоматизации GDB для динамического анализа бинарников
- В структуре полезной нагрузки: NOP-слайд, шеллкод, адрес возврата (`modules/generate_payload.py`)
- В принципах модульной архитектуры инструментов безопасности

Вся логика реализована на чистом Python с подробными комментариями. Сторонние фреймворки не используются — только стандартная библиотека и базовые утилиты Unix.

## Требования

Для работы необходимы следующие инструменты в `PATH`:

| Инструмент | Назначение | Установка (Debian/Ubuntu) |
|------------|------------|---------------------------|
| Python 3.8+ | Основная среда | `sudo apt install python3` |
| GDB | Динамический анализ бинарников | `sudo apt install gdb` |
| Binutils (`as`, `objcopy`) | Ассемблирование шеллкода | `sudo apt install binutils` |
| util-linux (`setarch`) | Отключение ASLR | `sudo apt install util-linux` |

Команда для установки всех зависимостей:
```bash
sudo apt update && sudo apt install python3 gdb binutils util-linux
```

## Использование

### Базовый пример (шеллкод execve)
Создать эксплоит для выполнения `/bin/ls`:
```bash
python3 main.py ./vulnerable_binary -s execve -d "/bin/sh" -o output/
```

### Вывод строки (шеллкод write)
Создать эксплоит, выводящий "Hello World":
```bash
python3 main.py ./vulnerable_binary -s write -d "Hello World" -o output/
```

### Аварийное завершение с segfault
Сгенерировать минимальную нагрузку для вызова segmentation fault:
```bash
python3 main.py ./vulnerable_binary -s crash -o output/
```

### Проверка шеллкода
Собрать автономный исполняемый файл для тестирования шеллкода:
```bash
python3 main.py ./vulnerable_binary -s execve -d "/bin/sh" --test-shellcode
```

### Справочник аргументов

| Аргумент | Описание | По умолчанию |
| :--- | :--- | :--- |
| `binary` | Путь к уязвимому 64-битному ELF-файлу | *(обязательно)* |
| `-s, --shellcode` | Тип шеллкода: `write`, `execve`, `crash` | `execve` |
| `-d, --data` | Данные для шеллкода (строка или команда) | *(обязательно для write/execve)* |
| `-o, --output-dir` | Директория для выходных файлов | `./output` |
| `-v, --verbose` | Включить подробное логирование | `False` |
| `--test-shellcode` | Создать тестовый бинарник вместо эксплоита | `False` |
